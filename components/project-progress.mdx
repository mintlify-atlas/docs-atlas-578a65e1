## Overview

The `ProjectProgress` component displays GitHub issue completion progress with a visual progress bar. It shows the number of completed tasks versus total tasks and renders a customizable progress indicator.

## Interface

### ProjectProgress Props

<ParamField path="text" type="string" required>
  Header text displayed above the progress bar (e.g., "Project Progress").
</ParamField>

<ParamField path="issues" type="Issue[]" required>
  Array of issue objects to track. Each issue must conform to the `Issue` interface.
</ParamField>

### Issue Interface

<ParamField path="title" type="string" required>
  The title of the issue.
</ParamField>

<ParamField path="isCompleted" type="boolean" required>
  Whether the issue is completed (closed).
</ParamField>

## Real-World Usage with GitHub API

The component is designed to work with GitHub Issues. Here's how it's used in the actual app:

```tsx
"use client";
import { fetchIssues } from "@/app/lib/actions";
import { useFetchIssuesInInterval } from "@/app/lib/hooks/useFetchIssuesInInterval";
import { Issue, ProjectProgress } from "@/components/project-progress";
import { GithubIssue } from "@/lib/types";

const githubIssuesToIssues = (githubIssue: GithubIssue) => {
  const issue: Issue = {
    title: githubIssue.title,
    isCompleted: githubIssue.state === "closed",
  };
  return issue;
};

export default function ProgressPage() {
  // Fetch issues every 10 seconds
  const issues = useFetchIssuesInInterval(10000);

  if (!issues) {
    return "No Issues";
  }

  return (
    <div className="flex h-screen w-full items-center justify-center">
      <ProjectProgress text="Project Progress" issues={issues} />
    </div>
  );
}
```

## Visual Display

The component renders:

1. **Header Text**: Large text (text-5xl) displaying the `text` prop
2. **Progress Bar**: A custom `Progress` component showing completion percentage
3. **Completion Text**: Shows "X / Y Tasks completed" below the bar

## Technical Details

### Calculation

The component calculates completion percentage:
```typescript
const completedIssues = issues.filter((issue) => issue.isCompleted);
const completedText = `${completedIssues.length} / ${issues.length} Tasks completed`;
```

## Use Cases

- **Stream Overlays**: Display project progress during coding streams
- **GitHub Milestones**: Track milestone completion in real-time
- **Sprint Progress**: Visualize sprint task completion
- **Goal Tracking**: Monitor any set of boolean completion states

## Integration with GitHub

This component is designed to work seamlessly with the GitHub API:

1. Fetch issues using GitHub's REST or GraphQL API
2. Map `state: "closed"` to `isCompleted: true`
3. Pass the mapped issues array to the component
4. Optionally use polling (like `useFetchIssuesInInterval`) for live updates
