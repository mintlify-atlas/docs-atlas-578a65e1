## Get Started in 5 Minutes

This quick start guide will have you streaming with your Progress Overlay in no time. Follow these streamlined steps to go from zero to a working overlay.

<Note>
  Already completed installation? Skip to [Step 4](#view-your-overlay-pages) to start using your overlay.
</Note>

## Setup Steps

<Steps>
  <Step title="Clone and Install">
    Clone the repository and install dependencies:
    
    <CodeGroup>
    ```bash npm
    git clone https://github.com/christian1koch/progress-overlay.git
    cd progress-overlay
    npm install
    ```
    
    ```bash yarn
    git clone https://github.com/christian1koch/progress-overlay.git
    cd progress-overlay
    yarn install
    ```
    
    ```bash pnpm
    git clone https://github.com/christian1koch/progress-overlay.git
    cd progress-overlay
    pnpm install
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Configure GitHub Token">
    Create a `.env.local` file with your GitHub token:
    
    ```bash
    echo "GITHUB_TOKEN=your_token_here" > .env.local
    ```
    
    <Warning>
      Need a GitHub token? Create one at [github.com/settings/tokens](https://github.com/settings/tokens) with `repo` scope.
    </Warning>
  </Step>
  
  <Step title="Set Your Repository">
    Edit `app/lib/actions.ts` to point to your GitHub repository:
    
    ```typescript app/lib/actions.ts
    export async function fetchIssues() {
        const octokit = new Octokit({
            auth: process.env.GITHUB_TOKEN,
        });

        const { data } = await octokit.rest.issues.listForRepo({
            owner: "your-username",      // Change this
            repo: "your-repo-name",      // Change this
            state: "all",
        });

        const issues = data.map(githubIssuesToIssues);
        return issues;
    }
    ```
    
    <Info>
      Use your GitHub username and the repository name you want to track.
    </Info>
  </Step>
  
  <Step title="Start the Development Server">
    Launch your overlay application:
    
    <CodeGroup>
    ```bash npm
    npm run dev
    ```
    
    ```bash yarn
    yarn dev
    ```
    
    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>
    
    Your overlay will be available at `http://localhost:3000`
  </Step>
</Steps>

## View Your Overlay Pages

Progress Overlay provides four distinct pages, each designed for different streaming scenarios:

### 1. Name Marquee - `/name`

**URL**: [http://localhost:3000/name](http://localhost:3000/name)

Displays your streamer name in an interactive curved marquee:

```tsx app/name/page.tsx
<CurvedLoop
    marqueeText=" ♥ Chris__Codes"
    speed={2.2}
    curveAmount={200}
    interactive
    className="fill-pink-500!"
/>
```

**Features**:
- Curved text path animation
- Interactive drag to change direction
- Customizable speed and curve amount
- Perfect for stream intros or transitions

<Info>
  **Customize it**: Edit `app/name/page.tsx` to change the `marqueeText` to your streamer name.
</Info>

### 2. ASCII Art Background - `/background`

**URL**: [http://localhost:3000/background](http://localhost:3000/background)

Creates stunning ASCII art effects with Three.js:

```tsx app/background/page.tsx
<ASCIIText 
    text="Chat!" 
    textColor="oklch(65.6% 0.241 354.308)" 
/>
```

**Features**:
- 3D text rendered as ASCII characters
- Wave animations and shader effects
- Interactive rotation following mouse movement
- Color gradient effects with hue rotation

<Info>
  **Customize it**: Change the `text` prop to display any message you want. Great for chat interactions!
</Info>

### 3. Project Info - `/project-info`

**URL**: [http://localhost:3000/project-info](http://localhost:3000/project-info)

Shows your GitHub repository link and all issues as checkboxes:

**Features**:
- Scrolling marquee with repository information
- Completed issues shown with checkmarks and strikethrough
- Open issues displayed as unchecked boxes
- Auto-refreshes every 10 seconds
- Displays issues sorted by ID

**What it shows**:
```
[✓] Implement authentication (completed)
[✓] Add user dashboard (completed)
[ ] Create settings page (open)
[ ] Add export functionality (open)
```

### 4. Progress Bar - `/project-info/progress`

**URL**: [http://localhost:3000/project-info/progress](http://localhost:3000/project-info/progress)

Displays a large animated progress bar:

```tsx components/project-progress.tsx
<ProjectProgress 
    text="Project Progress" 
    issues={issues} 
/>
```

**Features**:
- Large, prominent progress bar visualization
- Completion percentage calculation
- "X / Y Tasks completed" counter
- Auto-refreshes every 10 seconds
- Custom colors and styling

**Example output**:
```
Project Progress
[████████████░░░░░░░░] 60%
6 / 10 Tasks completed
```

## Add to OBS Studio

Now that your overlay is running, add it to OBS:

<Steps>
  <Step title="Add Browser Source">
    In OBS Studio:
    1. Click the **+** button in the Sources panel
    2. Select **Browser**
    3. Name it (e.g., "Progress Bar Overlay")
  </Step>
  
  <Step title="Configure Source">
    In the browser source properties:
    
    - **URL**: Enter the full URL (e.g., `http://localhost:3000/project-info/progress`)
    - **Width**: 1920 (or your stream resolution width)
    - **Height**: 1080 (or your stream resolution height)
    - **FPS**: 30
    - ✅ Check **Shutdown source when not visible**
    - ✅ Check **Refresh browser when scene becomes active**
  </Step>
  
  <Step title="Position and Resize">
    - Drag the overlay to your desired position
    - Resize using the red corner handles
    - The overlay has a transparent background by default
  </Step>
</Steps>

<Info>
  For detailed OBS setup instructions and advanced configuration, see the full [OBS Setup Guide](/guides/obs-setup).
</Info>

## Customize Your Overlay

### Change Refresh Interval

Both `/project-info` and `/project-info/progress` pages use the `useFetchIssuesInInterval` hook with a 10-second interval:

```typescript
const issues = useFetchIssuesInInterval(10000); // 10 seconds in milliseconds
```

To change the refresh rate, edit the value in:
- `app/project-info/page.tsx`
- `app/project-info/progress/page.tsx`

### Modify Visual Styles

All components use Tailwind CSS for styling. Edit the component files to customize:

- **Progress Bar Colors**: `components/project-progress.tsx`
- **Text Styles**: Any component's `className` props
- **ASCII Effect Colors**: `app/background/page.tsx` - change `textColor`
- **Name Marquee Curve**: `app/name/page.tsx` - adjust `curveAmount` and `speed`

### Update Text Content

Customize displayed text:

```tsx
// Name marquee
marqueeText=" ♥ YourName"

// ASCII background
text="Your Message"

// Progress bar title
text="Your Project Name"
```

## Understanding the Data Flow

Here's how Progress Overlay fetches and displays your GitHub data:

<Steps>
  <Step title="Server Action Fetches Data">
    The `fetchIssues()` server action in `app/lib/actions.ts` uses Octokit to call the GitHub API:
    
    ```typescript
    const { data } = await octokit.rest.issues.listForRepo({
        owner: "christian1koch",
        repo: "progress-overlay",
        state: "all",  // Fetches both open and closed issues
    });
    ```
  </Step>
  
  <Step title="Hook Polls for Updates">
    The `useFetchIssuesInInterval` hook calls the server action at regular intervals:
    
    ```typescript
    export function useFetchIssuesInInterval(intervalInMs: number) {
        const [issues, setIssues] = useState<Issue[]>();
        
        useEffect(() => {
            const fetchData = async () => {
                const newIssues = await fetchIssues();
                setIssues(newIssues);
            };
            const intervalId = setInterval(fetchData, intervalInMs);
            return () => clearInterval(intervalId);
        }, []);
        
        return issues;
    }
    ```
  </Step>
  
  <Step title="Data Transformation">
    GitHub issues are transformed into a simpler format:
    
    ```typescript
    export function githubIssuesToIssues(githubIssue: GithubIssue) {
        const issue: Issue = {
            id: githubIssue.id,
            title: githubIssue.title,
            isCompleted: githubIssue.state === "closed",
        };
        return issue;
    }
    ```
  </Step>
  
  <Step title="Components Render Data">
    React components display the transformed issue data with visual elements:
    - Checkboxes for task status
    - Progress bars for completion percentage
    - Sorted and filtered lists
  </Step>
</Steps>

## Common Customizations

### Add More Repository Information

Edit the marquee items in `app/project-info/page.tsx`:

```tsx
<Marquee
    items={[
        "Your Project Name",
        "♣",
        "Github: username/repo-name",
        "♣",
        "Follow me on Twitch!",
        "♣",
    ]}
/>
```

### Change Progress Bar Colors

Modify colors in `components/project-progress.tsx`:

```typescript
const GREY_AZURE = "#510424";  // Background color
const CYAN = "oklch(65.6% 0.241 354.308)";  // Indicator color
const LIGHT_GREY = "#f7f7f7";  // Text color
```

### Track Multiple Repositories

To track multiple repos, you can:

1. Duplicate the `fetchIssues` function with different repo parameters
2. Create new page routes for each repository
3. Combine data from multiple repositories in a single view

## Production Deployment

For production use (not localhost), deploy to Vercel:

<Steps>
  <Step title="Install Vercel CLI">
    ```bash
    npm install -g vercel
    ```
  </Step>
  
  <Step title="Deploy">
    ```bash
    vercel
    ```
  </Step>
  
  <Step title="Set Environment Variables">
    In the Vercel dashboard, add your `GITHUB_TOKEN` to environment variables
  </Step>
  
  <Step title="Update OBS URLs">
    Replace `localhost:3000` with your Vercel URL (e.g., `https://your-app.vercel.app/project-info/progress`)
  </Step>
</Steps>

<Warning>
  When deploying to production, ensure your GitHub token is set as an environment variable in your hosting platform, not committed to your repository.
</Warning>
